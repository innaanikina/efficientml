### Проделанные эксперименты

#### Эксперимент 1. Реализация triton кернеля для квантизации матрицы весов

Составлен понедельный план реализации проекта. Подготовлен Dockerfile, настроено cuda-совместимое окружение. Реализован triton кернель для квантизации 2D матрицы весов из fp16 в int4 и тесты для него. Кернель устроен следующим образом:

Сначала читаем pytorch тензор из fp16 чисел, упакованных row-based (строка1:строка2:строка3 и тд). Каждая строка читается в своем кернеле со своим program_id, то есть в отдельном cuda-блоке. Это позволяет значительно распараллелить и ускорить процесс квантизации. Затем в цикле считываем по BLOCK_SIZE элементов и таким образом находим глобальный абсолютный максимум во всей строке. Далее во втором цикле выполняем симметричную квантизацию: делим w на абсолютный максимум и умножаем на 7, затем приводим w к int. Таким образом, веса w оказываются в диапазоне от -7 до 7 и их можно хранить в int4. Так как большинство видеокарт не имеют нативной поддержки хранения int4 и операций с ними, то полученные числа необходимо упаковать по 8 штук в одно число типа int32 (это и есть выбранная схема упаковки - первое число храним в первых четырех битах int32, второе - в следующих и т. д.). С одной стороны, такая упаковка имеет недостаток, так как для работы с полученным квантизованным тензором всегда необходимо выполнять распаковку и это занимает время. С другой стороны, ожидается, что значительный выигрыш в памяти, занимаемой матрицей весов W (fp16 -> int4 - около 4 раз; не ровно 4 раза, так как нужно сохранять еще и абсолютные максимумы для деквантизации) приведет к значительному ускорению загрузки этой матрицы из HBM (High Bandwidth Memory) в SRAM (Static Random Access Memory) так как современные LLM являются именно IO-bound.

Запуск бенчмарка с кернелем для квантизации:
```python
python triton_kernels/quantize_kernel.py
```
Результаты для матриц 64x64, 256x256, 1024x1024, 4096x4096, веса семплируются из стандартного нормального распределения:
 - Сжатие минимально для матрицы 64x64 (в 3.74 раза) и приближается к сжатию в 4 раза (3.98 для 1024x1024) так как с увеличением длины строки доля в памяти одного fp16 для хранения абсолютного максимума для этой строки снижается.
 - Ошибка квантизации увеличивается с ростом размера матриц, так как с увеличением размерности строки увеличивается вероятность выброса в этой строке, что приведет к увеличению абсолютного максимума и как следствие к расширению бинов и большей плотности чисел в одном бине. При квантизации и деквантизации:
 - MSE: для 64x64 0.011, для 4096x4096 0.024
 - MAE: для 64x64 9%, для 4096x4096 14%
 - Max Abs Error: для 64x64 30%, для 4096x4096 38%.


### Запуск контейнера

Dockerfile совместим с драйверами cuda 12.3 или новее

В docker-compose.yaml можно пробросить желаемый порт (для triton-vis, например) или добавить команду запуска jupyter notebook

```bash
docker compose up -d
```


### Запуск тестов

```bash
pytest -sv
```
